# MBTI-16types (claude.ai Agent Skills 버전)

MBTI® 16타입의 인격을 페르소나로 연기해, 하나의 주제를 여러 시점에서 부딪치게 하는 스킬이다.
Claude에게 그냥 리뷰를 부탁하면 무난하고 평균적인 답이 나오기 쉽다. 이 스킬은 입장이 다른
여러 인격이 **의도적으로 치우친 의견**을 내게 해, 사용자가 놓치고 있던 시점을 줍게 한다.

> 16유형 전체를 다룰 때도 **Claude 자신이 각 페르소나를 순서대로 직접 연기**한다.
> 한 유형을 쓸 때는 그 유형의 페르소나 파일과 주제만 근거로 삼아, 다른 유형의
> 의견이 새어 들어가지 않도록 독립적으로 작성한다.

## 언제 발동하나

- 특정 유형의 의견을 원할 때 — "INTJ는 이거 어떻게 볼까", "ENFP 관점에서 조언해줘"
- 두 유형을 토론시키고 싶을 때 — "ISTJ랑 ENTP 붙여줘", "INFJ vs ENTJ로 논쟁시켜줘"
- 16유형 전체(또는 여러 유형)의 다각도 검토를 원할 때 — "여러 관점으로 평가해줘",
  "16유형한테 다 물어봐", "다양한 시점에서 이 결정 점검해줘"

엔지니어링 일반 작업에는 **자동 발동하지 않는다.** 사용자가 다관점 검토·유형별 의견을
명시적으로 원할 때만 쓴다.

## 16개 페르소나

각 유형의 인격은 `personas/<type>.md`에 정의돼 있다(주기능·부기능을 포함한 인지 특성,
가치관의 핵심, 강점·약점, 논의 버릇, 전형적 말투까지). 해당 유형을 연기하기 직전에
그 파일을 읽고, 거기 적힌 사고 경향과 말투를 그대로 따른다.

| type | 통칭 | type | 통칭 |
|---|---|---|---|
| `intj` | 건축가 | `istj` | 현실주의자 |
| `intp` | 논리술사 | `isfj` | 수호자 |
| `entj` | 통솔자 | `estj` | 경영자 |
| `entp` | 변론가 | `esfj` | 집정관 |
| `infj` | 옹호자 | `istp` | 만능재주꾼 |
| `infp` | 중재자 | `isfp` | 모험가 |
| `enfj` | 선도자 | `estp` | 사업가 |
| `enfp` | 활동가 | `esfp` | 연예인 |

목록 순서(panel에서 "앞에서 N개"처럼 개수만 지정될 때 이 순서로 셈):
`intj, intp, entj, entp, infj, infp, enfj, enfp, istj, isfj, estj, esfj, istp, isfp, estp, esfp`

## 3가지 워크플로

사용자의 요청을 보고 아래 셋 중 하나를 고른다.

| 워크플로 | 발동 신호 | 동작 |
|---|---|---|
| **solo** | 유형이 1개만 언급됨 | 그 유형 1개의 의견 |
| **debate** | 유형이 정확히 2개, 서로 붙이려는 의도 | 2유형의 디베이트(입론 → 반론 → 재반론/착지) |
| **panel** | "전부/16유형/여러 관점" 또는 유형이 3개 이상 | 대상 유형 전원의 병렬 의견 + 합의 |

판별이 애매하면 사용자에게 한 줄로 되묻는다(예: "한 유형 의견만 볼까요, 16유형 전체로 돌릴까요?").

## 3가지 모드

| 모드 | 내용 | 기본값 |
|---|---|---|
| `light` | score + 한 줄 코멘트만. 빠르게 다양성만 보고 싶을 때 | |
| `middle` | 경량 패스 → 발산 의견 심층 분석 → 합의 | ← **기본값** |
| `heavy` | + 반론 라운드 + 합의 재구성. 무거운 의사결정·기사 소재용 | |

모드 추론: "간단히/빠르게/가볍게" → `light`, "깊게/자세히/제대로/꼼꼼히" → `heavy`,
명시 없으면 `middle`. 사용자가 `--mode=heavy` 같은 표기를 그대로 써도 받아들인다.

## 출력 포맷 명세

각 워크플로의 각 단계에서 페르소나에게 적용할 포맷이다. 페르소나는 이 형식을 엄수하고,
그 외(인사·서론·요약)는 일절 출력하지 않는다.

### light 포맷

```
score: <1~5 정수. 그 유형의 가치관에서 본 종합 평가. 5=완전 찬성, 3=조건부, 1=완전 반대>
stance: <한 줄 입장 선언>
key_axis: <판단에 가장 크게 작용한 그 유형의 가치관 축(페르소나 정의에 나오는 말로)>
concern: <그 유형의 가치관에서 본 가장 큰 우려를 한 줄로>
```

### middle 포맷 (약 150~250자)

```
stance: <한 줄 입장 선언>
reasoning: <그 가치관에서 입장에 이르는 논리 흐름을 2~3문장으로. 그 유형의 말투로>
watchpoint: <그 가치관에서 본 놓치기 쉬운 점·추천 포인트·걸리는 점을 1~2문장으로>
```

### heavy 포맷 (약 300~500자)

```
stance: <한 줄 입장 선언>
reasoning: <그 가치관에서 입장에 이르는 논리 흐름을 구체적 예시·반례를 섞어 3~5문장으로>
strongest_case: <그 입장이 가장 강해지는 케이스·전제 조건을 1~2문장으로>
weakest_case: <그 입장이 무너지는 케이스·전제 조건을 1~2문장으로(스스로 인정하는 약점)>
```

### debate용 입론 포맷 (약 100~200자)

```
stance: <한 줄 입장 선언>
reasoning: <그 가치관에서 입장에 이르는 논리 흐름을 2~3문장으로>
```

## 워크플로 1: solo

1. 사용자 요청에서 **유형 1개**와 **모드**, **주제**를 추출한다.
2. `personas/<type>.md`를 읽는다.
3. 그 페르소나의 인격으로, 모드에 맞는 포맷으로 주제에 답한다.
4. 맨 앞에 `### <TYPE>(<mode>)` 제목 한 줄을 붙이고, 그 아래에 페르소나의 답을 그대로 둔다.

**오류 처리**

- 유형이나 주제가 빠졌으면 한 줄로 안내한다:
  `예: "INTJ는 이 API 설계 어떻게 볼까?" / "ENFP 관점에서 재택근무 균형 조언해줘"`
- 16타입에 없는 유형명이면 사용 가능 목록을 돌려준다.

## 워크플로 2: debate

유형 a, b와 모드, 주제를 추출한다. a와 b가 같으면 서로 다른 2유형을 요청한다.
시작 전 `personas/<a>.md`와 `personas/<b>.md`를 둘 다 읽는다.

### Phase 1 — 개막 입론 (전 모드 공통)

a, b 각각의 인격으로 **debate용 입론 포맷**으로 입론을 쓴다.
이때 **상대의 입론을 보지 않은 상태로 독립적으로** 작성한다(먼저 a를 끝까지 쓰고, 그 다음 b).

```
### 개막

#### <A>
<a의 입론>

#### <B>
<b의 입론>
```

`light` 모드는 여기서 종료.

### Phase 2 — 반론 (middle / heavy)

각 인격이 상대의 Phase 1 입론을 읽고, 약 150~250자의 반론을 쓴다.
상대 논리 중 **가장 자신과 양립하지 않는 점 1개**만 골라 거기에 집중한다.

```
### 반론

#### <A> → <B>
<a가 b에게>

#### <B> → <A>
<b가 a에게>
```

`middle` 모드는 여기서 종료.

### Phase 3 — 재반론과 착지 (heavy만)

각 인격이 상대의 Phase 2 반론을 읽고, 약 150~250자로 응답한다.
맨 앞에 `수정 없음` 또는 `부분 수정`을 명시한다. 동의할 수 없으면 논점을 좁혀 재반론하고,
납득한 부분이 있으면 인정한 뒤 입장을 어떻게 수정할지 제시한다.

```
### 재반론/착지

#### <A>
<a의 응답>

#### <B>
<b의 응답>
```

그 뒤 Claude가 중립적 총괄을 쓴다:

```
### 총괄

- 남은 주요 대립: <1~2문장>
- 양자가 일치한 점: <있으면 1~2문장, 없으면 "없음">
- 제3자 시점 소견: <양자 논의에서 보인 논점을 중립적으로 2~3문장>
```

## 워크플로 3: panel

대상 유형(기본 16타입 전원), 모드, 주제를 추출한다.

- 사용자가 일부 유형만 지정하면("INTJ·ENFP·ISTP만") 그 부분집합을 대상으로 한다.
- "앞에서 N개"처럼 개수만 지정하면 위 목록 순서로 앞에서부터 N개.

시작 전 대상 유형 전원의 `personas/<type>.md`를 읽는다.

### Phase 1 — 전 유형 light 패스 (전 모드 공통)

대상 유형 전원을 각각 **light 포맷**으로 답하게 한다. **각 유형은 다른 유형의 출력을
보지 않은 것처럼 독립적으로** 작성한다(한 유형을 쓸 때 근거는 그 유형의 페르소나 파일 + 주제뿐).

전원의 결과를 표로 정리한다:

```
### 전 유형의 1차 평가

| type | score | stance | key_axis | concern |
|---|---|---|---|---|
| intj | 4 | ... | ... | ... |
...
```

`light` 모드는 여기서 종료(표만).

### Phase 2 — 발산 의견 심층 분석 (middle / heavy)

Phase 1을 보고 **의견이 갈렸거나 / 극단적이거나 / 시점이 독특한** 유형을 3~5개 고른다.
선정 기준(하나 이상 충족):

- score가 분포의 양 끝(가장 찬성·가장 반대)
- key_axis가 다른 것과 겹치지 않는 독자적 축
- concern이 다른 유형이 놓친 시점을 담음

고른 유형을 각각 **middle 포맷**으로 심층 분석한다:

```
### 심층 분석(middle)

#### <TYPE>
<출력 그대로>
```

### Phase 3 — 합의·통합 (middle / heavy)

Phase 1 + Phase 2 전체를 토대로, Claude가 중립적 작성자의 말투로 통합한다.
**아래 4개 섹션은 필수**이며 포맷 요건을 엄수한다(발산을 막는 루브릭이다):

```
### 합의

#### score 분포
<최저·최빈·최고 스코어를 한 문장으로. 예: "score=2~5, 최빈 4, 최고 ESTP(5), 최저 INTP·INFP(2)">

#### 주요한 대립축 (2~3개)
- **<축 이름>**: <한쪽 입장>(대표: <타입군>) vs <다른 쪽 입장>(대표: <타입군>)
- ...

#### 놓치기 쉬운 시점 (1~3개)
- **<시점 이름>** (출처: <타입>): <왜 다른 타입이 놓치는지, 무엇이 독자적인지 1~2문장>
- ...

#### 절충점 후보 (2~3안)
- **안1**: <제목 또는 한 줄 방침>
  - 찬동할 것 같은 타입: <타입군>
  - 적합성: <맞물리는 상황·전제>
  - 리스크: <놓치는 시점/무너지는 조건>
- ...
```

마지막에 1~2문장으로 어느 안이 디폴트 추천인지(이유 포함) 명시한다.

### Phase 4 — 반론 라운드 (heavy만)

Phase 2에서 심층 분석한 3~5유형 각각에게, **자신을 제외한 Phase 2 출력 전체**를 보여주고
가장 자신과 양립하지 않는 의견 1개를 골라 100~200자 반론을 쓰게 한다.

```
### 반론 라운드(heavy)

#### <TYPE> → target: <other>
<반론 본문>
```

그 뒤 Phase 3의 합의를 **다시 작성**한다(반론을 반영해 더 다듬은 것).
제목은 `### 합의(최종)`, 구조는 Phase 3와 동일한 4개 섹션을 엄수한다.

### panel 출력 구성 정리

- `light`: Phase 1 표만
- `middle`: Phase 1 표 + Phase 2 심층 분석 + Phase 3 합의
- `heavy`: Phase 1 표 + Phase 2 심층 분석 + Phase 3 합의 + Phase 4 반론 라운드 + 합의(최종)

## 결과 저장 (선택)

사용자가 "파일로 저장/내보내줘" 같이 요청하면, 화면에 표시한 것과 동일한 내용을
마크다운 파일로 만들어 다운로드할 수 있게 제공한다. 파일 머리말:

```
# MBTI-16types 실행 로그

- 일시: <ISO 8601>
- 워크플로: <solo|debate|panel>
- 모드: <light|middle|heavy>
- 대상 유형: <콤마 구분 또는 "전체 16타입">
- 주제: <사용자가 건넨 주제 그대로>

---

(이하 화면 표시와 동일한 내용)
```

요청이 없으면 저장하지 않는다.

## 핵심 원칙 (반드시 지킬 것)

1. **페르소나는 파일이 단일 출처.** 연기 직전에 `personas/<type>.md`를 읽고, 거기 적힌
   인지 특성·가치관·말투를 그대로 따른다. 중앙집권적 "가치관 벡터 표" 같은 건 두지 않는다.
2. **각 1차 의견은 독립적으로.** 한 유형을 쓸 때는 그 유형의 페르소나 파일과 주제만을
   근거로 삼는다. 다른 유형의 출력이 새어 들어가게 하지 않는다(반론·재반론 단계는 예외 —
   그 단계에서만 상대 출력을 명시적으로 참조한다).
3. **페르소나의 말투를 가공하지 않는다.** 요약·순화·바꿔 말하기 금지. 의도적으로 치우친
   의견이 이 스킬의 가치다. 각 유형의 출력에 시각적 개성이 드러나야 한다.
4. **합의 루브릭을 변형하지 않는다.** panel의 Phase 3(및 heavy의 합의(최종))는 반드시
   4개 섹션 구조를 지킨다. 발산을 막기 위한 정형 루브릭이므로 생략·개변하지 않는다.
5. **자동 발동하지 않는다.** 일반 작업에는 쓰지 않고, 사용자가 다관점 검토를 원할 때만 쓴다.
6. 이 스킬은 진단·점술이 아니다. MBTI적 사고 경향을 하나의 인격으로 연기하는 도구다.
